Indirect3D

This may well crash your browser! You probably don't want to try to run the animation in an instance where you're doing anything important.

The animation is two triangles, defined as a set of vertexes, each with a position and a primary color with 50% alpha. One triangle is fixed in space while the other one moves around it.

Only runs in Firefox 4 for some reason. Chromium doesn't give any errors but doesn't work, either.

To start, hit Enter. To stop, hit Esc.

Known issues

  • Coloring algorithm for triangles is just plain wrong.
  • Something is not transforming correctly, at all.
    • The moving triangle should rotate in place.
    • Also the bottom right corner should really be the top.
    • When I order the transforms the way I think I should, things get very weird.
    • Moving the camera doesn't quite do what you thing it will.
  • No triangle culling yet, which leads to...
  • ...aberrant behavior with triangles of certain orientations.
  • It's slow as hell, but I don't think that will change.

Things that work pretty well

  • The alpha-blending algorithm seems basically accurate.
  • Coloring algorithm for lines is spot on, at least for low-distortion projections.
